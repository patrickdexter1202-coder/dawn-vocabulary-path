# 上海六年级英语学习系统 Spec

> 历史版本：当前完整规格请以 `english-learning-system-spec-v0.7.md` 为准。

> 版本：V0.1（方案草案）  
> 日期：2026-06-29  
> 目标阶段：一期——词汇学习与暑假打卡；二期——语法等能力扩展  
> 目标用户：上海地区、即将进入或正在就读六年级、英语基础一般的学生  
> 文档用途：产品评审、内容准备、交互设计、技术拆分与验收的共同基线

## 1. 结论摘要

1. 系统长期词库规模为 **2,000–3,000 个去重词条**，但不要求学生在一个暑假内全部新学。2026 年暑假建议完成约 800–1,200 个词的诊断或复现，并把其中约 **430–600 个薄弱词/新词学稳**。
2. 一期以“上海教材顺序 + 国家课程标准范围”为主轴：国家课标决定基础范围，学生所在学校实际使用的教材决定学习顺序，通用高频词表只用于补充。
3. 初学阶段采用高提示、低挫败的引导模式；熟练后逐步切换到无提示主动回忆和快速混合练习。系统自动升降级，不要求学生自己理解复杂算法。
4. 采用“间隔学习 + 主动提取 + 及时反馈”的自适应复习机制。艾宾浩斯遗忘思想作为设计背景，不照搬一套对所有学生相同的固定日期表。
5. 2026 年上海中小学暑假为 **7 月 1 日至 8 月 31 日，共 62 天**。每天都可以打卡，但一周安排 5 个完整学习日、1 个复习日、1 个轻量日，不把连续 62 天满负荷学习作为成功标准。
6. 核心成功指标不是“点过多少词”或“连续打卡多长”，而是 **经过 7–30 天仍能主动回忆的掌握词数**。

## 2. 背景、假设与边界

### 2.1 已知背景

- 上海采用五四学制，六年级处于初中起始阶段。
- 目标学生小学英语基础一般，需要同时补小学词汇漏洞和预习六年级词汇。
- 学生年龄通常不足 14 周岁，账号、学习行为和语音等数据均需按儿童个人信息进行审慎设计。

### 2.2 V0.1 默认假设

- 学生为 **2026 年暑假后升入六年级**；若已在六年级，计划起点由诊断结果自动后移。
- 每天可投入 18–22 分钟，家长只承担首次授权、时间设置和每周查看报告，不负责每天陪学。
- 具体学校和教材版本尚未给出，因此教材必须是可配置项，不能把“牛津上海版”或某一旧版硬编码进系统。
- 一期只详细设计词汇；句型和语法只保留可扩展框架。

### 2.3 一期不做

- 不承诺一个暑假记完 2,000–3,000 个新词。
- 不把连续打卡、排行榜或奖励兑换做成学习的主要目的。
- 不以只认中文释义作为“掌握”；掌握至少覆盖词义回忆、声音辨认和基本拼写中的核心维度。
- 不在没有版权授权时抓取或复制教材全文、商业词典释义、例句和音频。
- 不在 P0 版本保存儿童原始语音；语音识别如需加入，另做隐私影响评估。

## 3. 产品目标与成功标准

### 3.1 一期目标

- 找出学生在小学基础词中的真实漏洞，而不是从 `a/an` 开始机械重学。
- 用低挫败路径建立每天学习的习惯，并逐步训练主动回忆能力。
- 让复习量由遗忘风险决定，避免“今天只顾新词、明天欠下一大堆复习”。
- 让学生看到可理解的进步，让家长看到可信但不过度监控的结果。
- 建立带来源、版本和版权状态的内容底座，后续可扩展到语法、听力和阅读。

### 3.2 暑假建议目标

以下为产品默认目标，可由诊断结果调整：

| 指标 | 基础路线 | 熟练路线 |
| --- | ---: | ---: |
| 62 天内有效打卡 | ≥ 54 天 | ≥ 54 天 |
| 完整学习日平均时长 | 18–22 分钟 | 18–25 分钟 |
| 深度学习的薄弱/新词 | 约 430–440 个 | 约 500–600 个 |
| 14 天延迟主动回忆率 | ≥ 80% | ≥ 85% |
| 30 天延迟主动回忆率 | ≥ 75% | ≥ 80% |
| 小学基础词诊断/复现覆盖 | 以 505 个课标二级词为锚点，叠加教材词 | 同左 |

这里的“深度学习”不等于首次展示过，而是至少跨 3 个自然日完成多次提取，并通过一次 7 天以上的无提示检查。

### 3.3 北极星指标与护栏指标

**北极星指标：每周新增的“耐久掌握词数”**，即在首次学习至少 7 天后仍通过无提示主动回忆的词数。

护栏指标：

- 复习按时完成率；
- 14/30 天延迟回忆率；
- 单次学习时长；
- 连续错误次数和使用提示比例；
- 因错误后直接退出的会话比例；
- 需要家长介入才能完成的天数；
- 复习债务规模。

不能仅用累计点击词数、总在线时长或连续打卡天数评价效果。

## 4. 词库与权威来源设计

### 4.1 目标规模与分层

词库以 **lemma（词元/基础词形）** 去重，短语单独计数，避免把 `go/goes/went/gone` 当作四个独立单词夸大词量。

| 层级 | 内容 | 用途 | 预计规模 |
| --- | --- | --- | ---: |
| L0 前置能力 | 字母、常见字母音、数字、星期等极基础项目 | 只给诊断暴露明显缺口的学生 | L1 的子集 |
| L1 小学基线 | 课标二级 505 词 + 上海实际小学教材补充词 | 巩固小学、填漏洞 | 约 505–800 |
| L2 六年级优先 | 学校选用的六年级上册单元词及必要短语 | 2026 暑假预习主线 | 约 250–450 |
| L3 初中基础 | 课标三级 1,600 词中除二级词外的 1,095 个基本词 | 六至九年级长期主线 | 1,095 |
| L4 扩展高频 | 与上海教材主题、分级阅读和常见生活场景相关的高频词 | 将长期词库扩展至 2,000–3,000 | 约 400–1,400 |

层级规模存在重叠，最终以去重后的有效词条数为准。课标另允许各地在 1,600 词基础上增加 100–300 词；这些词应进入本地扩展层，不应假装都是全国统一必学词。

### 4.2 来源优先级

“收哪些词”和“先学哪些词”需要分开治理：

1. **国家范围锚点**：教育部《义务教育英语课程标准（2022 年版）》三级词汇表，共 1,600 词，其中二级词汇 505 个、初中阶段基本词汇 1,095 个。
2. **上海教学顺序**：上海市教委当学期教学用书目录及学生学校实际选用的教材。2026 年春季目录已出现上海教育出版社“五四学制·英语”六年级下册教学参考资料；2026 年秋季目录发布后仍需做一次版本复核。
3. **教材单元词表**：取得合法授权的教材词汇、短语、音频和单元标签。教材负责排序和校内同步，不能仅凭市场上流传的词表导入。
4. **释义与发音**：优先采购或授权成熟学习词典数据；也可由教研团队原创适龄释义、例句并双人复核。Oxford 3000/5000、Cambridge English Vocabulary Profile 等只能作为扩展选词和难度映射参考，使用前确认许可。
5. **扩展高频层**：使用可合法利用的儿童/分级阅读语料频率，并通过教研审核，不能因为“高频”就压过教材必学词。

### 4.3 单词内容模型

每个词条至少包含：

- `lemma`、英/美拼写变体、常见屈折变化；
- 适龄中文核心义项，义项之间分开建模；
- 词性、音标或音素、英音/美音音频；
- 一个六年级可理解的短例句和中文提示；
- 可选图片、词根词缀、音节/拼读分块；
- 易混词、同反义词、常用搭配；
- 课标层级、教材版本、年级、册次、单元、主题标签；
- 来源证据、内容版本、版权/授权状态、审核人和审核日期；
- 难度、具体/抽象程度、是否适合图片表达等教学属性。

同一单词的不同义项不能被一个笼统中文翻译掩盖；一期可只教当前教材所需的核心义项，后续按螺旋方式增加。

### 4.4 内容质量门禁

- 新内容至少经过“教研编辑 + 第二审核人”两人复核。
- 自动检查重复词、错误词形、音频缺失、例句超纲、释义循环和版权字段缺失。
- 每次发布形成不可变 `content_version`，学生历史记录继续指向当时版本。
- 教材改版时做映射迁移，不直接覆盖旧记录。

## 5. 学习路径与四种背词模式

### 5.1 总体路径

```mermaid
flowchart LR
  A["权威来源与授权内容"] --> B["版本化词库"]
  B --> C["短诊断 + 学中诊断"]
  C --> D["个性化暑假计划"]
  D --> E["引导学习 / 主动回忆"]
  E --> F["答题事件与错误类型"]
  F --> G["自适应复习调度"]
  G --> D
  F --> H["学生进步与家长周报"]
```

### 5.2 初始诊断

- 首次诊断控制在 6–8 分钟、约 25–40 个自适应抽样题，不做一次性 2,000 词大考。
- 依次抽样词义识别、听音辨词和基础拼写；答错不公开排名、不出现负面评价。
- 输出四类状态：`已稳固`、`可能掌握`、`薄弱`、`未学`。
- 诊断只生成初始概率，之后通过日常学习持续修正，避免一次发挥决定整个暑假计划。

### 5.3 四种模式

| 模式 | 面向状态 | 主要形式 | 提示与反馈 | 退出条件 |
| --- | --- | --- | --- | --- |
| M0 认识它 | 陌生、连续答错 | 看图/情境、听音、英中二选一、跟读 | 高提示，错误后立即展示正确答案 | 同一会话完成一次辅助回忆 |
| M1 搭起来 | 初学、拼写薄弱 | 音节分块、缺字母、首字母提示、音形匹配、短语境选择 | 逐步撤掉提示，指出具体错误 | 跨 2 天核心题正确率 ≥ 75% |
| M2 想出来 | 基本熟悉 | 中文/图片到英文、听音拼写、英文到核心义、例句填空 | 先作答后反馈，可查看一次提示 | 7 天窗口无提示正确率 ≥ 80% 且提示率 < 20% |
| M3 快练 | 已熟练 | 无提示混合题、限量快速回忆、易混词对比 | 默认不显示倒计时压力，答后简洁反馈 | 14 天延迟回忆稳定；答错则自动回落 |

系统根据表现自动升降级。连续两次错误时立即增加支架，不让学生在“不会—继续猜—继续错”中消耗信心。

### 5.4 单词掌握定义

每个词分别维护 `meaning`、`listening`、`spelling/production` 和 `context` 四个维度，不用一个模糊总分掩盖短板。

一个词进入“耐久掌握”至少满足：

1. 在 3 个不同自然日中完成至少 4 次有效提取；
2. 至少一次为 7 天以上间隔后的无提示主动回忆；
3. 最近三次核心尝试中至少两次正确；
4. 至少通过一次词义主动回忆或听音拼写，而不只是选择题猜中。

“认识”与“会用”分开显示：一期的耐久掌握主要代表能稳定理解并基本产出，不宣称等同于熟练写作运用。

## 6. 记忆与复习算法

### 6.1 设计选择

可选方案：

| 方案 | 优点 | 问题 |
| --- | --- | --- |
| 固定“艾宾浩斯表” | 实现简单、容易解释 | 个体差异大；漏学后容易形成巨大债务 |
| 纯 SM-2 类算法 | 调度效率高、资料多 | 对低龄初学者较生硬；自评“难/易”不稳定 |
| **儿童友好的混合调度（推荐）** | 固定脚手架易理解，同时按正确率、提示和延迟表现适配 | 需要更完整的答题事件和规则测试 |

本系统选择第三种。艾宾浩斯的价值在于“遗忘随时间发生、复现要拉开间隔”，不是给每个孩子套一条精确相同的遗忘曲线。

### 6.2 两层复习循环

**同一会话内**：首次理解 → 2–5 分钟后辅助回忆 → 会话结束前再次提取。

**跨日默认锚点**：约 1、3、7、14、30 天；开学后的长期模式再加入 60、90 天。锚点是初始值，实际日期由表现调整。

调度信号：

- 是否正确；
- 是否使用提示；
- 反应时是否明显异常；
- 题型是识别还是主动回忆；
- 最近错误类型；
- 距离上次复习的真实间隔；
- 当前复习债务和当日可用时间。

### 6.3 结果处理规则

| 结果 | 系统动作 |
| --- | --- |
| 错误/放弃 | 当次切回 M0/M1；数分钟后再现；次日优先复习 |
| 使用关键提示后答对 | 记为“困难”，不算无提示掌握；下次间隔缩短 |
| 无提示正常答对 | 按默认锚点延长 |
| 无提示快速且跨维度答对 | 可适度延长，但不能跳过首次 7 天延迟检查 |
| 10 天内失败 ≥ 5 次 | 标为“顽固词”，更换图片、拼读、词族或语境，不再机械重复同一张卡 |

### 6.4 复习债务保护

- 每日顺序永远是：到期复习 → 薄弱词 → 新词。
- 基础路线每日复习上限建议为 60 词或 15 分钟，取先到者；系统造成的超额债务不能让学生失去打卡资格。
- 若到期词 > 40、近 7 天主动回忆率 < 75%，或预计会话超过 22 分钟，自动下调新词量，必要时当天不加新词。
- 回归学习时先生成 3–5 天恢复计划，不要求一天补完所有欠账。

## 7. 2026 年暑假计划与每日打卡

上海市教委校历明确 2026 年暑假从 7 月 1 日开始，8 月 31 日结束，共 62 天。

### 7.1 分阶段计划

| 阶段 | 日期 | 重点 | 基础路线新词量 |
| --- | --- | --- | ---: |
| A 启动 | 7/1–7/3 | 监护人授权、短诊断、熟悉 M0/M1 | 0–6/天 |
| B 找回基础 | 7/4–7/17 | 小学薄弱词优先，建立成功感 | 8/完整学习日 |
| C 稳定增长 | 7/18–8/7 | 小学漏洞 + 六上高优先词 | 12/完整学习日 |
| D 提升效率 | 8/8–8/23 | 逐步进入 M2/M3，混合新旧词 | 15/完整学习日 |
| E 收口 | 8/24–8/31 | 原则上停止大批加词，做 14/30 天检查和开学衔接 | 0–8/天，按债务决定 |

按照每周 5 个完整学习日估算，基础路线约可深度学习 430–440 个薄弱/新词；若近 14 天表现稳定，可每天增加 2–4 词，上限约 600。系统不得为了追进度突破当日时长与复习债务护栏。

### 7.2 每周节奏

- 第 1–5 天：新词 + 到期复习；
- 第 6 天：只复习、易混词挑战和小结；
- 第 7 天：5–8 分钟轻量日，只做少量到期词和听音回忆；
- 每 14 天：一次无提示小测和计划校准。

“每天打卡”不等于“每天加新词”。轻量日也能保持接触，同时给记忆巩固和家庭安排留出空间。

### 7.3 单日会话

完整学习日默认 18–22 分钟：

1. 3 分钟：到期词热身；
2. 8–10 分钟：新词理解与第一次提取；
3. 6–8 分钟：新旧混合主动回忆；
4. 1 分钟：今日结果与明日预告。

复习日 12–15 分钟，轻量日 5–8 分钟。完成全部系统上限内的核心任务即可提前结束，不鼓励为了在线时长挂机。

### 7.4 有效打卡规则

打卡分为三种且都是真实记录：

- **完整打卡**：完成当日系统给出的到期复习和计划新词；
- **复习打卡**：当日计划本就不加新词，完成核心复习队列；
- **轻量打卡**：完成至少 5 个到期词及 1 组听音回忆，通常用于每周轻量日或恢复期。

规则：

- 只打开应用、挂机或全部跳过不算有效打卡；
- 系统因复习债务主动缩减任务时，完成缩减后的队列仍算有效；
- 允许 48 小时内完成“补学”，但原日期标记为补学，不伪造成按时完成；
- 连胜中断不清空累计成果，主展示采用“本周 6/7 达标”和成长地图，连续天数只做次级信息；
- 家长可设置旅行/生病恢复日，不允许用惩罚、羞辱文案或公开排名催学。

### 7.5 开学模式

9 月起默认切换到：

- 周一至周四：8–12 个教材同步新词 + 到期复习；
- 周五：本单元复习；
- 周末：一天累计复习，一天轻量/休息；
- 考试周可设为“守成模式”，暂停新词，只保留高风险复习；
- 教材单元进度可由家长或教师调整，复习算法不因换单元而清零。

## 8. 用户角色与核心功能

### 8.1 角色

- **学生**：完成诊断、每日任务、复习和查看个人成长。
- **监护人**：授权账号、设置可学习时段、查看周报、调整目标强度和处理数据权利请求。
- **内容管理员/教研**：维护词库、来源、版本、版权和内容审核。
- **运营/支持**：查看匿名化产品健康指标，不默认查看单个儿童的完整答题明细。

### 8.2 P0 功能需求

| ID | 功能 | 关键验收标准 |
| --- | --- | --- |
| FR-01 | 监护人建档与授权 | 不满 14 周岁用户在监护人同意前不能开始收集个性化学习数据；可撤回并删除账号数据 |
| FR-02 | 教材与目标配置 | 可选择教材版本/未知版本、每天可用时长和暑假/开学模式；版本可后改且不丢历史 |
| FR-03 | 轻量诊断 | 6–8 分钟内完成；可暂停；输出分层结果和首周计划，不给公开分数排名 |
| FR-04 | 今日计划 | 明确展示预计时长、到期复习、新词量和打卡类型；复习债务高时自动降新词 |
| FR-05 | M0–M3 学习 | 至少支持英中理解、听音辨词、分块拼写、无提示回忆；连续错误自动增加支架 |
| FR-06 | 自适应调度 | 每次作答形成可追踪事件；能够根据正确、提示、题型和间隔生成下一复习时间 |
| FR-07 | 打卡与恢复 | 支持完整/复习/轻量三类；补学和按时完成分开记录；中断后生成恢复计划 |
| FR-08 | 学生进步页 | 显示耐久掌握词、本周目标、薄弱维度和成长轨迹，不用同龄公开排名 |
| FR-09 | 家长周报 | 每周只突出完成情况、延迟回忆、预计负担和建议；避免逐题监控式信息轰炸 |
| FR-10 | 内容后台 | 可导入、审核、版本化、回滚词条；未标来源或版权状态的内容不能发布 |
| FR-11 | 数据导出与删除 | 监护人可以查看、导出和删除儿童数据；删除流程有明确完成状态 |

### 8.3 交付形态方案

| 方案 | 优点 | 主要代价 | 建议 |
| --- | --- | --- | --- |
| 响应式 Web/PWA | 开发验证快、手机和电脑均可用、迭代成本低 | 系统提醒和微信家庭触达较弱 | **V0 原型推荐** |
| 微信小程序 | 上海家庭触达自然、打卡分享和监护人入口方便 | 平台审核、隐私合规和能力差异增加复杂度 | 完成 2–3 周真实试点后评估 |
| 原生 App | 离线、音频和语音能力最好 | 一期成本与维护负担最高 | 暂不推荐 |

推荐先用渠道无关的后端与数据模型实现 Web/PWA 核心闭环；验证孩子确实能独立完成后，再决定是否以小程序作为主入口。

## 9. 系统模块与核心数据

### 9.1 模块划分

1. **内容中心**：词条、义项、音频、教材标签、来源和版本。
2. **学习画像**：学生设置、初始诊断和每词多维掌握状态。
3. **计划引擎**：暑假阶段、教材优先级、新词配额和复习债务。
4. **学习会话**：M0–M3 题型、提示、反馈和错误恢复。
5. **调度引擎**：根据答题事件计算下一次复习。
6. **打卡服务**：任务完成、补学、轻量日和连续性展示。
7. **报告与实验**：学生进步、家长周报、匿名化产品指标和算法版本对比。
8. **监护与合规**：同意记录、最小化采集、导出、删除和审计。

### 9.2 核心实体

| 实体 | 关键字段 |
| --- | --- |
| `GuardianAccount` | 监护人身份、联系方式、同意版本、同意/撤回时间 |
| `LearnerProfile` | 昵称、学段、教材配置、每日时长、计划强度；避免采集非必要精确身份信息 |
| `Word` / `WordSense` | 词元、词形、义项、音频、例句、教学属性 |
| `SourceRef` | 课标/教材/词典来源、版本、页或单元、版权状态 |
| `CurriculumTag` | 地区、学制、年级、册次、单元、课标层级 |
| `UserWordState` | 四维掌握状态、当前模式、稳定度、难度、下次复习时间、顽固词标记 |
| `ReviewEvent` | 题型、提示、答案、正误、反应时、错误类型、发生时间、内容/算法版本 |
| `StudyPlan` | 日期范围、目标层、日时长、强度、教材优先级 |
| `DailyTask` | 到期词、新词、预计时长、任务类型、完成状态 |
| `CheckIn` | 原日期、完成日期、打卡类型、是否补学、有效性 |
| `ContentVersion` / `AlgorithmVersion` | 发布版本、变更说明、启停状态和迁移规则 |

`ReviewEvent` 是不可变事实记录；`UserWordState` 是可重算结果。这样算法升级时可回放历史，不会失去解释能力。

## 10. 非功能与儿童保护要求

- **最小化采集**：学生可用昵称；若只需判断年龄段，不采集精确生日、学校全名或班级。
- **监护人同意**：不满 14 周岁个人信息属于敏感个人信息，处理前取得父母或其他监护人同意，并提供拒绝、撤回、查看、更正和删除入口。
- **隐私默认值**：不公开排名、不默认分享打卡、不使用精确位置、不做面向儿童的行为广告。
- **语音**：P0 使用预置音频和本地跟读即可；若将来上传语音，须说明必要性、保存周期和删除机制。
- **可用性**：学生完成两次引导后，应能不依赖家长独立完成每日任务；按钮和文字适合触控，不以颜色作为唯一反馈。
- **性能**：普通家庭网络下今日任务首屏可用时间目标 < 2 秒，题目切换 < 300ms，音频点击后开始播放目标 < 500ms；弱网可缓存当日任务。
- **可靠性**：作答事件本地暂存并幂等上传，断网不重复扣进度；内容发布可回滚。
- **可解释性**：家长能看到“为什么今天新词变少/为什么这个词又出现”，但不暴露复杂公式。
- **合规审计**：正式上线前由法务/隐私负责人按当时有效法规完成专项复核，本 Spec 不是法律意见。

## 11. 二期能力框架

二期不另起一套系统，复用内容版本、学习事件、调度、打卡和报告能力。

### 11.1 语法

- 以 `GrammarPoint` 建立知识点和前置依赖，例如一般现在时依赖人称、be 动词和基础动词词形。
- 每个知识点采用：微讲解 → 例句观察 → 引导操练 → 变式辨析 → 句子产出 → 间隔复习。
- 建立错误类型，而不只记“对/错”，如主谓一致、时态选择、词序、冠词遗漏。
- 将学生已掌握词汇用于语法例句，降低同时处理生词和语法的认知负担。

### 11.2 其他可扩展模块

- 听力：音素/单词 → 短句 → 教材语篇；
- 阅读：分级短文、词汇覆盖率、上下文猜词；
- 口语：跟读、句型替换和情景表达，语音上传另做合规设计；
- 写作：句子拼装 → 仿写 → 短段落，并接入错误本；
- 错题本：跨词汇、语法、听读模块统一复习。

## 12. 验证与验收计划

### 12.1 原型可用性

- 先用 5–8 名目标年龄学生做任务观察，验证首次建档、诊断和完整学习日。
- 成功条件：第二次使用时 ≥ 90% 的学生能独立完成；中位完整会话 ≤ 22 分钟；关键操作无家长解释也能理解。

### 12.2 两周小规模试点

- 建议 12–20 名基础不同的学生；记录真实完成、提示、错误和退出原因。
- 用一组未在训练界面重复展示的等价题做 7/14 天延迟检查。
- 重点验证：新词量是否过高、选择题是否制造虚假掌握、复习债务是否失控、打卡是否变成家长任务。

### 12.3 暑假验收

- 产品侧：≥ 80% 的活跃学生没有出现连续 3 天以上的不可完成复习债务。
- 学习侧：达到基础路线的学生群体，14 天延迟主动回忆率中位数 ≥ 80%。
- 体验侧：完整学习日中位时长 18–22 分钟，因连续错误退出的会话 < 5%。
- 内容侧：已发布词条 100% 具有来源、版本和版权状态，抽检无严重释义/音频错误。

阈值需通过首轮试点校准，不能为了达成指标事后降低题目难度或把选择题正确冒充主动回忆。

## 13. 后续开发前必须确认的事项

1. 目标学生是 2026 年“升六年级”还是“升七年级”；
2. 所在学校 2026 秋季实际使用的英语教材和版本；
3. 日均可投入时间是否确实约 20 分钟；
4. 家长是否愿意完成首次授权并每周看一次报告；
5. 首发形态选择 Web/PWA 还是微信小程序；
6. 教材、词典释义、例句、图片和发音音频的授权来源；
7. P0 是否需要语音识别，还是只保留听音与自我跟读；
8. 是否有教师/教研人员参与首批词条审核。

其中第 2、6、8 项属于内容上线硬门槛，其余可通过原型试点验证。
