# 晨光词径产品规格 v0.3

> 历史版本：当前完整规格请以 `english-learning-system-spec-v0.8.md` 为准。

更新日期：2026-06-30  
适用阶段：上海五四学制，2026 年秋季升六年级，本地短期试运行

## 1. 核心变化

本版由“打开网页即看单词”调整为“每日学习会话”：

1. 先选择词库；
2. 点击“开始本次学习”；
3. 系统根据历史表现动态生成计划；
4. 每个词完成跟读与默写；
5. 会话结束后保存时间、成绩与错误明细；
6. 同一天可再次开始，重新动态配词。

## 2. 动态选词规则

基础计划为 30 词，按以下优先级去重组卷：

1. **活跃高频错词**：累计答错至少 2 次，且尚未连续答对 3 次；全部强制进入本次计划，不受 30 词上限影响。
2. **到期复习词**：`nextReviewAt` 已到期，按到期时间和薄弱程度排序。
3. **普通巩固词**：做过但未到期的词，按错误数、提示数和连续正确数排序，基础目标约占计划 40%。
4. **未学新词**：从当前词库补足计划，优先选择当天尚未安排过的词。
5. **补位词**：词库不足或当天多次学习时，再使用当天已经安排但尚未作答的新词或普通巩固词。

间隔复习周期为 1、3、7、14、30 天。使用提示后答对只安排 1 天间隔；答错后立即进入后续巩固候选。连续答对 3 次后，高频错词退出强制队列，但历史错误仍保留供家长查看。

## 3. 会话与每日记录

### 3.1 会话记录

每次开始创建独立 `Session`：

```text
Session {
  id, date, libraryId,
  startedAt, endedAt,
  plannedWordIds[], attempts[]
}
```

用户可完成全部计划，也可点击“结束本次学习”提前结束。结束后显示本次用时、作答数、答对数和答错数，并可再次开始。

### 3.2 作答明细

每次提交保存：词条 ID、英文、中文义、用户答案、对错、是否使用提示、提交时间。错误明细在“学习记录”的具体会话下展示，家长可查看每次错误单词、填写内容与时间。

### 3.3 统计层级

- 顶部：进行中显示“本次完成数 / 计划数”，始终显示“今日作答数、今日错词数”。
- 左侧：今日会话数、作答数、不同单词数、对错与提示；进行中增加本次进度。
- 学习记录：今日汇总 → 每次会话 → 本次错误明细。
- 错词巩固：历史错词、累计错误次数、当前待巩固/已稳定状态。

## 4. 默写防提示

进入默写输入阶段后，左侧英文、音标、中文释义、例句和来源整体模糊并覆盖提示卡，同时设置 `aria-hidden=true`，避免视觉和辅助技术泄露答案。提交后恢复显示。

默写卡仍显示当前中文题面，并可请求部分字母提示。

## 5. 界面细节

- 音标只显示文本，移除旁边的小播放按钮；保留右侧主播放按钮。
- 词库原生箭头改为内置图标，距选择框右边 19px。
- 用户姓名显示为“陆梵”。
- 首页、学习、默写、结果、会话完成、学习记录和错词清单均沿用既有晨光词径视觉语言。

## 6. 本地存储

新存储键：`dawn-vocabulary-progress-v2`。

```text
ProgressV2 {
  version: 2,
  sessions: Session[],
  wordStats: { [wordId]: WordStat },
  wrongWords: { [wordId]: WrongWordHistory }
}
```

系统会读取并迁移 v1 的每日记录和错词，旧数据不主动删除。所有数据仍只存在当前电脑、当前浏览器。

## 7. 验收标准

- 未开始时不显示具体单词；选择词库后才能开始。
- 高频错词强制进入后续会话；到期复习优先于新词。
- 默写阶段答案区域不可辨认，提交后恢复。
- 每次会话有开始、结束时间和独立汇总。
- 同日可开始多次，且新词尽量不重复。
- 家长能查看每次错误单词、用户答案和时间。
- v1 数据可迁移到 v2。
- 自动化测试、生产构建和本地迁移包验证通过。
