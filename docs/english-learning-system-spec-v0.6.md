# 晨光词径产品规格 v0.6

> 历史版本：当前完整规格请以 `english-learning-system-spec-v0.8.md` 为准。v0.7 暂缓原生 iOS 与多终端方向，v0.8 已上线 GitHub Pages + 浏览器本地数据。

更新日期：2026-07-02  
适用阶段：上海五四学制，2026 年秋季升六年级，本机短期试运行

## 1. 产品目标与当前边界

晨光词径用于家庭场景下的英语词汇跟读、默写和错词巩固。当前版本是本机 Web 应用，不含账号、云同步、语音识别评分或多设备同步。

当前初始内容包括：

- 小学基准词库：498 条；
- 可核实的六年级上册 Unit 1 学习词条：44 条；
- 默认“衔接混合”词库：六年级词优先，与小学基准按标准化词形去重合并。

“六年级词”必须具备教材版本、册次、单元和来源证据，不能仅凭“不在小学基准内”推断。2026 年秋实际教材仍需按学生实物教材、ISBN 和页码复核。

## 2. 当前学习流程

1. 学生选择词库并点击“开始本次学习”。
2. 系统按学习历史动态生成本轮词表，默认上限为 30 个，但候选不足时允许少于 30 个；高频错词可使本轮超过基准数量。
3. 每个词依次完成“听音/跟读 → 看中文默写英文 → 查看结果”。
4. 默写时，左侧单词、音标、释义和例句从视觉及可访问性树中隐藏。
5. 答错词进入去重后的巩固清单；连续答对达到退出条件后不再强制复习。
6. 每天可开始多次，记录每次开始、结束、逐词答案、提示使用和正误。

回车键在跟读阶段进入默写，在结果阶段进入下一词；输入框、按钮、链接、下拉框、菜单和弹窗不会被页面级快捷键截获。

## 3. 动态选词规则

候选优先级为：

1. 活跃高频错词，必须出现；
2. 已到期的活跃错词；
3. 其他活跃错词；
4. 尚未学习的新词。

没有错误历史的已学正确词不进入后续轮次，也不用于补足 30 个。候选组内部先随机打乱；存在其他选择时，尽量避免连续出现相同首字母。当前词库没有新词和待巩固错词时，不创建空会话。

## 4. 朗读策略

单词朗读采用双层策略：

1. 优先请求有道在线英音：`https://dict.youdao.com/dictvoice?audio=<word>&type=1`；
2. 在线音频在 3.5 秒内未开始、加载失败、播放失败或浏览器拒绝播放时，自动回退到浏览器/操作系统 `speechSynthesis`，语言为 `en-GB`；
3. 正常速度 1.0 倍、慢速 0.75 倍同时适用于两种播放方式；
4. 两种方式均失败时，页面显示网络或系统语音设置提示。

因此当前版本的网页与词库可本地运行，但高质量在线发音需要联网；离线时能否朗读取决于电脑是否安装可用的英文系统语音。

## 5. 学习记录与家长操作

- 记录首页按会话展示答对、答错数量；展开后显示本次涉及词语、提交答案、正确答案和正误。
- 错词巩固展示仍处于活跃状态的错误词。
- 删除一次会话或移除一个错词状态需要固定家长密码 `99bill`。
- 删除会话后重算派生统计；移除错词仅取消后续强制复习，不删除历史作答。

固定密码仅用于家庭本机防误操作，不是安全认证机制。

## 6. 当前数据模型与存储

学习数据保存在当前浏览器 `localStorage` 的 `dawn-vocabulary-progress-v2` 中，包含：

- `sessions`：会话时间、词库、计划词和逐词作答；
- `wordStats`：累计次数、正确/错误次数、连续正确和复习时间；
- `wrongWords`：错词状态、错误频次和是否已由家长移除。

系统兼容并自动迁移 `dawn-vocabulary-progress-v1`。清除浏览器站点数据会丢失记录，记录不会自动跨电脑或跨浏览器同步。

## 7. 词库导入规划（尚未实现）

词库导入在当前架构上可行。建议把内置词库保持为随版本发布的只读内容，把用户导入词库保存到 IndexedDB；学习记录仍兼容现有 v2，并通过稳定 `wordId` 引用自定义词条。

首期范围建议为 CSV、JSON 和 XLSX：选择文件与 Sheet、字段映射、预览校验、去重策略、事务导入、导入报告、词库启停/归档，以及“自定义词库 + 学习记录”备份导出。完整设计见 `vocabulary-import-design.md`。

## 8. iOS 规划（尚未实现）

推荐使用 Capacitor 将现有 React/Vite 应用封装为 iOS App，而不是首版完全重写 SwiftUI。首个可安装测试版预计 5–9 人日、约 8万–15万开发 Token；达到可提交 App Store 的家庭试用版本预计累计 10–18 人日、约 16万–30万 Token。估算前提和替代路线见 `ios-development-assessment.md`。

## 9. 当前质量基线

- 3 个自动化测试文件，共 26 项测试通过；
- Vite 生产构建成功；
- 已覆盖选词、记录、删除、回车操作、在线发音优先和本地语音回退；
- 桌面、平板和手机响应式规则保留，但 iOS 真机尚未验收。

## 10. 下一阶段验收重点

- 导入失败不得破坏既有词库或学习记录；
- 自定义词条必须有稳定 ID，词库归档后历史记录仍可解释；
- 支持完整数据备份和恢复后，才把“可长期本地使用”作为正式承诺；
- iOS 版本必须在真实 iPhone 上验证音频、持久化、键盘、安全区和后台恢复。
