# 晨光词径产品规格 v0.7

更新日期：2026-07-02  
适用场景：单一家庭、单个在线网页、浏览器本地数据

## 1. 当前产品边界

晨光词径是一个通过 GitHub Pages 在线访问的静态 Web 应用。家庭成员使用电脑或 iPhone 浏览器打开同一个网址学习，但当前阶段不建设账号、云数据库或多终端数据同步。

本阶段明确不做：

- 注册、登录、家庭账号和儿童账号；
- Supabase、Firebase、CloudBase 或自建服务器；
- 跨电脑、跨手机、跨浏览器自动同步；
- iOS 原生 App、IPA 安装包、Capacitor 或 SwiftUI；
- 服务器端保存学习行为和自定义词库。

GitHub Pages 只分发 HTML、CSS、JavaScript、字体和内置词库，不接收学习记录。

## 2. 在线访问与部署

生产站点使用 GitHub Pages，以 HTTPS 地址访问。源码和构建通过 GitHub 仓库管理，GitHub Actions 在主分支更新后自动执行测试、构建并发布 `app/dist`。

项目型 Pages 地址形如：

`https://<github-user>.github.io/<repository>/`

部署必须处理 Vite `base` 子路径、字体相对路径、静态资源缓存和发布后健康检查。GitHub Pages 网站默认公开可访问，不把网址本身视为身份认证或隐私保护。

## 3. 词库

### 3.1 内置词库

- 小学基准：498 条；
- 六年级上册 Unit 1：44 条可核实学习词条；
- 衔接混合：六年级词优先，与小学基准去重合并。

内置词库随网页版本发布，浏览器不能直接修改源数据。

### 3.2 自定义词库（已实现）

支持从 CSV、JSON、XLSX 导入。导入流程包括 Sheet 选择、字段映射、前 20 行预览、校验、去重策略、事务写入和结果报告。XLSX 使用按需加载的 ExcelJS，只读取工作表单元格显示文本，不执行公式或宏。

自定义词库保存在当前浏览器 IndexedDB 中，不上传 GitHub。刷新页面后仍然存在，但以下操作不会自动带走数据：

- 换手机或换电脑；
- 换 Safari、Chrome、Edge 等浏览器；
- 使用无痕浏览；
- 清除站点数据；
- 浏览器或系统回收网站存储。

因此词库导入同时提供带版本号的完整 JSON 备份与恢复功能。

## 4. 学习与选词

学习流程保持为“选择词库 → 听音跟读 → 看中文默写英文 → 查看结果 → 下一词”。默写时隐藏答案区域。

动态选词优先级：活跃高频错词、到期活跃错词、其他活跃错词、未学新词。没有错误历史的已学正确词不用于补足轮次。候选组随机打乱，并尽量避免连续相同首字母。

## 5. 朗读

优先访问有道在线英音；3.5 秒未开始、加载或播放失败时，回退到浏览器/系统英文语音。1.0 倍和 0.75 倍速度对两种播放方式都生效。

GitHub Pages 在线访问不代表有道音频一定成功；外部音频失败时仍须保持本地语音回退和明确错误提示。

## 6. 学习记录

学习记录继续保存在当前浏览器 `localStorage` 的 `dawn-vocabulary-progress-v2`：

- 每次学习的开始、结束和词库；
- 逐词提交答案、提示使用、正误和时间；
- 单词累计统计和连续正确；
- 错词巩固状态及家长移除标记。

记录与自定义词库均不上传 GitHub，也不会自动同步。完整备份把 `localStorage` 学习记录和 IndexedDB 自定义词库打包到同一个带版本号的 JSON 文件；恢复前显示备份时间、词库数和会话数，并明确提示会覆盖当前浏览器数据。

## 7. iPhone 移动网页

此前已经设计并实现响应式适配：

- `max-width: 920px`：平板/窄屏双列布局；
- `max-width: 680px`：手机单列布局；
- `max-width: 390px`：小屏细节收紧；
- 已有 390×844 视口 QA，按钮保留至少 44px 触控目标。

因此 iPhone 可以直接用 Safari 打开 GitHub Pages HTML 页面，无需安装 App。

但“已设计适配”不等于“真机全部验收”。发布前仍需在真实 iPhone Safari 检查：

- 顶部/底部安全区；
- 地址栏伸缩引起的视口高度变化；
- 软键盘遮挡默写输入框；
- 音频自动播放限制和系统语音；
- IndexedDB/localStorage 持久性；
- 添加到主屏幕后重新打开的数据和布局。

## 8. 数据风险与备份责任

当前架构以简单、低成本为优先，接受“每个浏览器各自一份数据”的限制。备份恢复是长期使用前的必做功能，不把浏览器存储描述为永久可靠存储。

建议页面显示最近备份时间，并在累计学习达到一定会话数、导入新词库或长时间未备份时提醒家长导出备份。

## 9. 验收标准

- GitHub Actions 自动测试和构建成功后才能发布；
- GitHub Pages 首页、字体、JS、CSS 均从项目子路径正确加载；
- 内置词库和浏览器历史升级后保持可用；
- 导入 1,000 词后刷新页面仍可选择该词库；
- 清空站点数据后，通过备份文件可恢复自定义词库和学习记录；
- 真实 iPhone Safari 完成一轮跟读、默写、记录查看和页面重开验证；
- 网页不产生任何云端学习数据请求。
