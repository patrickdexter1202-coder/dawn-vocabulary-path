# 晨光词径产品规格 v0.8

更新日期：2026-07-02  
适用场景：单一家庭、GitHub Pages 在线网页、每个浏览器一份本地数据

## 1. 已上线产品边界

正式网址：`https://patrickdexter1202-coder.github.io/dawn-vocabulary-path/`

晨光词径是 React/Vite 静态 Web 应用。GitHub Pages 只分发 HTML、CSS、JavaScript、字体和内置词库；学习记录与自定义词库不上传 GitHub。

当前不提供账号、云数据库、跨设备自动同步、服务器端学习数据、原生 iOS App 或 IPA。相同网址在不同浏览器中仍是不同的数据副本。

## 2. 内置默认词库

线上构建包含以下只读词库，无需导入即可学习：

- `衔接混合`：默认选择，542 条；六年级词优先，再合并小学基准并按英文去重；
- `小学基准`：498 条课标基准词；
- `六年级新词`：44 条可核实的六年级上册 Unit 1 学习词条。

内置词库编译在 `app/src/vocabulary.generated.js` 中，随网页版本发布。浏览器清除站点数据不会删除内置词库，但会删除本地学习记录和自定义词库。

## 3. 自定义词库

支持 CSV、JSON、XLSX：文件选择、Sheet 选择、字段映射、前 20 行预览、必填校验、重复策略、事务写入与结果报告。

必填字段为 `word`、`meaning`；当前保存的可选字段为 `answer`、`phonetic`、`example`、`unit`、`source`、`tags`。完整规则和模板见 [词库导入格式 v1](vocabulary-import-format-v1.md)。

自定义词库保存在 IndexedDB 数据库 `dawn-vocabulary-local-v1`。导入成功后立即出现在开始页词库选择中；刷新页面后仍可使用。归档只影响新会话，不删除历史会话快照。

限制：单文件 10 MB、任一 Sheet 20,000 行；XLSX 只读取显示文本，不执行公式或宏。

## 4. 学习流程与选词

学习流程：选择词库 → 跟读 → 看中文默写英文 → 结果 → 下一词。默写阶段从视觉与无障碍树隐藏答案区。

动态选词优先级：活跃高频错词、到期活跃错词、其他活跃错词、未学新词。已学且没有错误历史的正确词不用于补足轮次。候选组先随机打乱，并尽量避免连续相同首字母。

## 5. 朗读

优先请求有道在线英音；3.5 秒未开始、加载或播放失败时回退到浏览器/系统英文语音。1.0 倍和 0.75 倍速度对两条播放路径都生效。两者均失败时显示明确错误提示。

## 6. 学习记录

学习记录保存在当前浏览器 `localStorage` 的 `dawn-vocabulary-progress-v2`，包含：

- 会话开始、结束、词库与计划词条；
- 每次提交答案、提示使用、正误和时间；
- 单词累计统计、连续正确与错词巩固状态；
- 家长移除错词标记。

删除单次会话或错词状态需要本地家长密码 `99bill`。该密码只防误操作，不是账号或安全认证。

## 7. 完整备份与恢复

“导出完整备份”把 v2 学习记录和 IndexedDB 自定义词库写入一个 `schemaVersion: 1` 的 JSON 文件。恢复前显示备份时间、词库数和会话数，并明确提示覆盖当前浏览器数据。

页面显示最近导出时间；导入新词库后提示立即备份。换设备、换浏览器、无痕模式、清除站点数据或系统回收网站存储前，家长必须主动导出备份。

## 8. 在线部署

- 公开仓库：`https://github.com/patrickdexter1202-coder/dawn-vocabulary-path`；
- 分支：`main`；
- 工作流：`.github/workflows/deploy-pages.yml`；
- 每次推送运行锁定依赖安装、38 项测试、Vite Pages 构建断言和官方 Pages 部署；
- Vite `base` 自动使用仓库子路径；品牌字体使用约 534 KB 的 WOFF2 字符子集，入口资源已完成线上验证。

## 9. 响应式与 iPhone

保留 `920px`、`680px`、`390px` 响应式断点、至少 44px 触控目标、`100dvh` 和安全区内边距。390×844 自动化/浏览器视口已验收；今日会话与作答摘要在手机端采用紧凑双列布局。

真实 iPhone Safari 的软键盘、音频限制、后台恢复、添加到主屏幕及长期存储生命周期尚未完成家庭设备验收，因此不宣称原生 App 级兼容。
