# 晨光词径 Web V0.1 实施计划

> 历史版本：当前状态请以 `implementation-plan-v0.8.md` 为准。

## 目标

在 `app/` 中实现与选定视觉稿一致的响应式学习页。首期只提供单词发音播放、正常/慢速切换、学生自我跟读确认和进入下一词；不采集或保存语音。

## 任务 1：创建原型工程

- 文件范围：`app/`。
- 行为：用 Product Design 原型模板创建 React/Vite 工程，保留模板的工程约定。
- 验证：依赖安装成功，初始测试与构建可运行。

## 任务 2：先锁定交互行为

- 测试文件：`app/src/App.test.jsx`。
- 先写并运行失败测试：页面显示 60 分钟四轮计划；播放按钮调用浏览器语音合成；速度切换更新为 0.75 倍；“我跟读好了”切换到下一个词；末词后完成本轮并回到首词。
- 最小实现文件：`app/src/App.jsx`、`app/src/words.js`。
- 通过方式：相关测试全部通过，测试观察用户可见行为而非内部状态。

## 任务 3：复刻晨光词径视觉

- 文件范围：`app/src/App.jsx`、`app/src/App.css`、`app/src/index.css`、`app/public/`。
- 行为：复刻 1440×1024 的柔和晨光渐变、左侧四轮进度、中心词条、右侧播放与跟读操作；中文使用东方大楷，图标使用选定图标库；提供键盘焦点、44px 触控目标和减少动态效果支持。
- 响应式：桌面保持三列构图；平板改为两列；手机将进度压缩为横向四段并把播放区移到词条下方。
- 验证：桌面、平板和手机不溢出，按钮状态清晰，核心文本对比度满足 WCAG AA。

## 任务 4：构建、预览与设计 QA

- 命令：运行测试、生产构建并启动本地预览。
- 证据：在 1440×1024 捕获默认状态，与选定视觉稿置于同一比较图中检查字体、间距、色彩、图标、文案和视觉层级。
- 修复门槛：清除全部 P0/P1/P2 差异；在项目根生成 `design-qa.md` 且 `final result: passed`。
