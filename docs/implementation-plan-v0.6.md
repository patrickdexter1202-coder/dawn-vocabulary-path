## 2. 下一阶段 A：词库导入（推荐先做）

### A1. 数据访问层

- 新建 `app/src/vocabularyStore.js` 和数据库版本常量；
- 先写 IndexedDB 建库、升级、事务回滚和刷新持久化失败测试；
- 建立内置词库与自定义词库统一查询接口，保持现有 `POOLS` 调用方可逐步迁移。

### A2. CSV/JSON 导入

- 新建解析与校验模块，先覆盖缺列、重复、非法类型和稳定 ID 测试；
- 实现字段映射、预览、重复策略和事务写入；
- 导入成功后可在现有词库下拉框选择。

### A3. 备份与恢复

- 定义带 `schemaVersion`、导出时间、词库和进度的 JSON 包；
- 覆盖往返一致性、旧版本升级、损坏文件不落库测试；
- 增加家长入口中的导出、导入和清空确认。

### A4. XLSX 与维护

- 增加 XLSX 解析依赖并记录许可证；
- 支持 Sheet 选择、列映射、行号错误报告；
- 增加词库改名、停用、归档，禁止破坏有历史记录的词条 ID。

### A5. 验收

- 运行完整自动化测试与生产构建；
- 手工导入 1,000 词 CSV、含重复和空行的 XLSX、损坏 JSON；
- 刷新浏览器、归档词库、备份清空再恢复，核对历史会话。

预计：MVP 4–7 人日/5万–9万 Token；完整本地长期版 8–14 人日/10万–18万 Token。

## 3. 下一阶段 B：iOS 可安装试用版

前置：确认 Mac、iPhone、分发目标，并完成或冻结 Web 端数据模型。

### B1. Capacitor 工程

- 配置 Vite 构建目录、iOS bundle identifier 和开发/生产环境；
- 在 Xcode 模拟器和真机加载现有应用。

### B2. iOS 适配

- 覆盖安全区、状态栏、软键盘、音量/静音、页面恢复；
- 接入原生语音回退或验证 WKWebView `speechSynthesis` 的可用性；
- 确保联网失败仍可继续学习和默写。

### B3. 数据与文件

- 验证应用升级后的持久化；
- 若词库导入已上线，接入 iOS 文件选择器，并将长期数据落入 SQLite 或可靠原生存储；
- 增加 Web 数据到 iOS 数据模型的一次性迁移测试。

### B4. QA 与发布选择

- 小屏 iPhone、主流 iPhone、iPad 各完成一轮；
- 家庭自用则交付签名测试包；需要外部分发则继续 TestFlight/App Store 准备。

预计：真机试用版 5–9 人日/8万–15万 Token；App Store 家庭试用版累计 10–18 人日/16万–30万 Token。

## 4. 推荐顺序

1. 先实现词库 IndexedDB 数据层与 JSON 备份；
2. 再完成 CSV/XLSX 导入；
3. 冻结数据格式后接入 Capacitor；
4. iOS 真机验证通过后，再决定是否投入 App Store 发布和原生重写。

这样可以避免 Web 和 iOS 分别设计一套词库迁移格式，也能让 iOS 首版直接复用已经经过真实数据验证的存储边界。
