# 晨光词径实施计划 v0.7

更新日期：2026-07-02

## 1. 目标

将现有本机 React/Vite 应用发布到 GitHub Pages；开发浏览器本地词库导入、完整备份和恢复；完成真实 iPhone Safari 验收。不开发登录、云数据库、跨终端同步或原生 iOS App。

## 2. 阶段 A：GitHub Pages 部署

### A1. 构建路径

- 更新 `app/vite.config.mjs`，通过环境变量设置 GitHub Pages 项目子路径；
- 将字体和其他根路径资源改为兼容 `base` 的相对/构建资源；
- 先添加构建产物路径断言，确保 HTML 不再引用错误的 `/fonts` 根路径。

### A2. 自动发布

- 新建 `.github/workflows/deploy-pages.yml`；
- 安装锁定依赖、运行 26 项现有测试、执行 Vite 构建；
- 使用 GitHub 官方 Pages Actions 上传并发布 `app/dist`；
- 仅允许主分支和手动触发部署。

### A3. 上线验收

- 检查首页和静态资源 HTTPS 状态；
- 完成跟读、默写、记录和刷新持久化冒烟测试；
- 验证有道失败时本地语音回退；
- 记录正式 URL、仓库可见性和回滚方法。

预计 0.5–1.5 人日，约 1万–2.5万 Token。

## 3. 阶段 B：本地词库导入

### B1. IndexedDB 数据层

- 新建词库、词条和数据库元数据对象仓库；
- 先测试建库、升级、事务回滚、稳定 ID 和归档语义；
- 将内置词库与自定义词库统一为页面可查询的数据接口。

### B2. CSV/JSON 导入

- 实现 UTF-8 CSV 与 JSON 解析；
- 完成字段映射、前 20 行预览、必填校验和重复策略；
- 事务成功后加入现有词库选择，失败不留下半套数据。

### B3. XLSX

- 选择 Sheet、映射列、只读取单元格显示值；
- 显示行号级错误报告；
- 记录解析依赖许可证，不执行宏或公式。

预计 CSV/JSON MVP 4–7 人日/5万–9万 Token；含 XLSX、维护和稳健错误处理累计 8–14 人日/10万–18万 Token。

## 4. 阶段 C：完整备份与恢复

- 定义 `schemaVersion`、导出时间、学习记录、词库和词条的 JSON 包；
- 实现导出、导入预览、覆盖确认和损坏文件拒绝；
- 恢复在一个受控流程中写入 localStorage 与 IndexedDB，任一阶段失败则回滚或保留原数据；
- 显示最近备份时间和周期性提醒。

预计 2–4 人日，约 3万–6万 Token；建议与词库导入同一阶段完成。

## 5. 阶段 D：iPhone Safari

- 为顶部和底部加入 `env(safe-area-inset-*)`；
- 评估 `100dvh` 与现有 `100vh` 的兼容降级；
- 输入框聚焦时滚动到软键盘上方；
- 在 375×667、390×844、430×932 视口回归；
- 使用真实 iPhone 验证音频、存储、刷新、后台恢复和添加到主屏幕。

预计代码适配 1–3 人日、约 2万–5万 Token；真机验收时间取决于设备可用性。

## 6. 推荐执行顺序

1. 先部署现有版本到 GitHub Pages，获得稳定测试网址；
2. 用该网址完成 iPhone 真机基础验收；
3. 开发 IndexedDB 词库导入；
4. 同步完成统一备份与恢复；
5. 再做完整桌面和 iPhone 回归。

## 7. 明确不进入本计划

- Supabase、Firebase、CloudBase；
- 登录、账号和权限；
- 自动跨设备同步；
- iOS 安装包及 App Store；
- 服务端保存或分析儿童学习数据。
