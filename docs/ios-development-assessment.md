# 原生 iOS 版本开发评估（暂缓）

更新日期：2026-07-02  
状态：历史路线评估，v0.8 不执行

> 当前边界仅要求 iPhone Safari 访问 GitHub Pages HTML 页面，不制作 IPA、不使用 Capacitor 或 SwiftUI。本文件保留供未来重新立项参考，不属于当前实施计划。

## 1. 结论与推荐

现有应用是 React 19 + Vite 的响应式单页应用，学习算法和本地数据逻辑已经独立，适合用 Capacitor 封装为 iOS App。首版推荐 Capacitor，而不是 React Native 或 SwiftUI 全量重写。

推荐分两步：先做可安装的真机试用版，验证音频、本地数据和 iPhone 布局；再决定是否补齐 App Store 发布、原生存储和更深的 iOS 体验。

## 2. 三条路线比较

### 路线 A：PWA/添加到主屏幕

- 复用程度接近 100%；支持主屏幕入口、全屏化和离线静态资源；
- 不是 App Store 原生安装包，系统集成和持久化控制较弱；
- 估算：1–3 人日，约 2万–4万 Token。

适合快速在家庭 iPhone 上试用，不适合作为正式 App 交付目标。

### 路线 B：Capacitor 封装（推荐）

- 复用 React 页面、CSS、选词算法和大部分测试；
- 建立 iOS Xcode 工程，用 WKWebView 加载构建后的网页；
- 补安全区、键盘、横竖屏、音频会话、应用图标和启动页；
- 首版可保留现有存储，正式版建议把自定义词库和关键进度迁到 SQLite/原生持久层，并提供一次性迁移；
- 可通过插件接入原生语音、文件选择、分享和备份。

估算：

- 可安装真机试用版：5–9 人日，约 8万–15万 Token；
- 可提交 App Store 的家庭试用版：累计 10–18 人日，约 16万–30万 Token；
- 若同时完成词库导入、SQLite 数据迁移和 iCloud 备份：累计 16–28 人日，约 25万–45万 Token。

### 路线 C：SwiftUI 原生重写

- 原生体验、离线语音、文件和持久化控制最好；
- UI、状态管理、选词、记录、导入和测试均需重新实现；
- Web 与 iOS 将形成两套界面代码，需要长期双端维护；
- 估算：25–45 人日，约 35万–70万 Token。

若再加入云同步、家长账户和完整内容后台，预算还会继续上升。当前阶段不推荐原生重写。

## 3. Capacitor 工作拆分

1. 工程接入（0.5–1.5 日）：安装 Capacitor、配置 `webDir`、创建 iOS 工程和 bundle identifier。
2. 页面适配（1–2 日）：安全区、状态栏、软键盘、iPhone 小屏和 iPad 分屏。
3. 音频（1–2 日）：验证有道 HTTPS 音频；失败时优先调用 iOS 原生英文语音，处理静音键、耳机和后台切换。
4. 数据（1–3 日）：验证 Web 存储生命周期；正式版设计 SQLite/文件备份及旧数据迁移。
5. 真机 QA（1–2 日）：至少覆盖一台较小 iPhone、一台主流 iPhone 和 iPad/模拟器。
6. 发布准备（2–5 日）：图标、启动页、隐私说明、签名、TestFlight、商店元数据和问题修复。

## 4. 前置条件与外部成本

- 必须有可运行受支持 Xcode 的 Mac；Windows 本机不能独立完成最终 iOS 构建、签名和真机调试。
- 需要 iPhone 真机做音频、键盘、持久化和生命周期验收，模拟器不能替代全部验证。
- 通过 App Store/TestFlight 分发需要 Apple 开发者相关账户、标识符和签名配置。
- 有道音频是外部在线依赖，正式发布前需再次评估接口稳定性、使用条款和替代源；本地/原生语音必须保持可用。

Apple 官方说明 Xcode 用于 Apple 平台的开发、测试和分发，并提供模拟器；Capacitor 官方说明其可以直接加入现有现代 JavaScript 项目，并通过原生容器和插件访问 iOS 能力。

## 5. Token 估算口径

上述 Token 包含需求细化、代码、自动化测试、构建问题排查和常规返工；不包含大范围视觉重设计、Apple 审核等待及不可预测整改、硬件和账户费用、教材内容人工校对，以及云后端/账号/同步。

建议预留 25% 风险缓冲。若只追求家庭内真机运行，按 8万–15万 Token 立项；若目标是 App Store 可发布且包含词库导入，建议按 25万–45万 Token 做总体预算。

## 6. 决策检查点

1. 目标是家庭自用真机版，还是 App Store/TestFlight 分发；
2. 是否已有 Mac、iPhone 和 Apple 开发者账户；
3. iOS 首版是否必须同时包含自定义词库导入。

在这三项确认前，建议先完成 Web 端词库导入的数据模型，使同一套词库和进度结构可被 iOS 复用。
