# 移动端学习流程紧凑化实施计划

## 目标

- 学习页不再展示例句区，避免无效信息占据首屏。
- 在 390 × 844 的典型手机视口中，尽量同时呈现单词、中文释义、朗读按钮和“我跟读好了”按钮。
- 默写输入与答案反馈尽量停留在当前视口，减少提交前后的上下滚动。
- 桌面端保持现有信息层级和舒展间距；所有主要触控按钮继续满足至少 44px 的触控高度。

## 实施

1. 在 `App.jsx` 移除例句节点，并为学习、默写、结果三个阶段增加状态类名。
2. 在移动端把本次学习进度压为一行，隐藏重复的今日统计明细。
3. 跟读阶段缩小单词、音标、朗读按钮、速度控件及纵向间距，隐藏非关键辅助说明。
4. 默写和结果阶段隐藏无意义的答案遮蔽占位，仅保留紧凑的默写卡片或结果卡片。
5. 先新增失败测试，覆盖例句移除、阶段类名和移动端关键密度规则，再完成实现。

## 验收

- 运行全部 Vitest 测试和生产构建。
- 使用 390 × 844 视口检查跟读、默写、结果三个状态。
- 确认跟读首屏包含单词、释义、朗读与下一步按钮；默写输入和提交后结果不因冗余占位产生大段滚动。
