# 词库导入与本地存储设计

更新日期：2026-07-02  
状态：v0.7 已实现  
边界：仅保存于当前浏览器，不做云端同步

## 1. 结论

可以建立词库导入功能，而且不需要后端服务器。推荐采用“内置词库 + IndexedDB 自定义词库”的双层模型：内置词库继续随代码发布，自定义词库在浏览器本地独立保存。`localStorage` 适合现有小规模进度数据，不适合承载可增长、需查询和事务更新的词库。

## 2. 方案比较

### 方案 A：全部放入 localStorage

实现最快，但容量、事务、查询和故障恢复能力都弱；大词库和 Excel 导入容易产生整块 JSON 重写。仅适合一次性原型，不推荐。

### 方案 B：IndexedDB（推荐）

浏览器原生支持结构化本地数据、索引和事务，无需启动额外服务。需要增加数据访问层和升级机制，但在 GitHub Pages 静态网页边界内长期成本最低。

### 方案 C：本地 SQLite/桌面服务

一致性和工具能力最好，但会把当前纯网页变成需要额外运行时的程序；超出 GitHub Pages + 浏览器本地数据的当前边界，不采用。

## 3. 首期支持格式

- CSV：最低依赖、易制作，必须支持 UTF-8；
- JSON：用于完整备份、恢复和程序化生成；
- XLSX：支持选择 Sheet 和字段映射，适合现有教材词表工作流；不执行公式或宏，只读取单元格显示值。

建议模板字段：

- 必填：`word`、`meaning`；
- 推荐：`phonetic`、`example`、`grade`、`book`、`unit`、`source`；
- 可选：`answer`、`tags`、`notes`、`sourceUrl`、`license`。

`answer` 缺省时等于 `word`。列名支持中英文别名，但导入前必须让用户确认映射。

## 4. 数据模型

`Library`：`id`、`name`、`description`、`grade`、`book`、`semester`、`source`、`license`、`contentVersion`、创建/更新时间和归档时间。

`VocabularyEntry`：`id`、`libraryId`、`word`、`normalizedWord`、`answer`、`meaning`、`phonetic`、`example`、`unit`、`tags`、`sourceRef`、启用状态和创建/更新时间。

`id` 使用稳定 UUID，学习记录始终引用此值。同一词库内默认以 `normalizedWord + meaning` 判断完全重复。相同英文但释义、单元不同的词可以保留，由用户在预览页决定合并、覆盖、跳过或新增。

## 5. 导入流程

1. 选择文件；XLSX 先选择 Sheet。
2. 自动识别表头并让用户确认字段映射。
3. 展示前 20 行预览和总行数。
4. 校验必填项、长度、空行、非法类型和重复项。
5. 用户选择重复处理策略。
6. 在一个 IndexedDB 事务中写入词库和词条；任一关键错误则整体回滚。
7. 显示导入报告：成功、跳过、合并、失败及具体行号。
8. 返回词库选择页，新词库默认可选但不自动开始学习。

## 6. 与学习历史的关系

- 导入后不改变既有会话和统计。
- 词条更新不得改变 `id`；修改拼写时需明确提示可能影响历史解释。
- 已产生历史记录的词条禁止硬删除，只能停用或归档。
- 词库归档后不参与新会话，但历史记录仍显示当时词形和释义快照。
- 导出备份必须同时包含自定义词库、学习记录、数据版本和导出时间。

## 7. 安全与容错

- 建议限制单文件 10 MB、单次 20,000 行；阈值应可配置。
- 不执行 Excel 宏、公式、HTML 或导入文本中的脚本。
- 页面渲染继续使用 React 文本节点，不使用未经清洗的 `innerHTML`。
- 数据库升级前生成可下载备份；导入事务失败不留下半套词库。
- 外部来源的教材词表仍需记录来源和授权状态，导入能力不等于内容已获授权。

## 8. 实施分期与估算

### P1：CSV/JSON 最小版

词库列表、模板下载、字段校验、预览、去重、IndexedDB 保存、选择学习、JSON 备份恢复。预计 4–7 人日，约 5万–9万开发 Token。

### P2：XLSX 与高级维护

Sheet 选择、字段映射、行级错误报告、覆盖/合并策略、词条编辑、归档。增加 2–4 人日，约 2.5万–5万 Token。

### P3：稳健长期版

数据库迁移、导入回滚测试、大文件性能、完整备份校验、冲突审计。累计 8–14 人日，约 10万–18万 Token。

Token 是工程生成与调试预算区间，不等同于 API 账单承诺；实际消耗受交互返工、源文件异常和浏览器兼容问题影响。

## 9. 验收标准

- 导入一个 1,000 词 CSV/XLSX 后刷新页面，词库和词条仍存在；
- 重复策略结果与预览一致；无效文件不会修改数据库；
- 自定义词库可参与现有随机、错词和未学词选词逻辑；
- 归档词库不会破坏历史会话；
- 导出后清空本地数据，再导入备份可恢复词库与进度；
- Chrome/Edge 桌面版通过，并补充真实 iPhone Safari 验收；不安排 WKWebView 或原生 iOS 验收。
