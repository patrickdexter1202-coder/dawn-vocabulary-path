# 词库导入与本地存储设计

更新日期：2026-07-02  
状态：v0.8 已实现；当前格式以 `vocabulary-import-format-v1.md` 为准
边界：仅保存于当前浏览器，不做云端同步

## 1. 结论

词库导入已在无后端条件下实现，采用“内置词库 + IndexedDB 自定义词库”的双层模型：内置词库随代码发布，自定义词库在浏览器本地独立保存。`localStorage` 继续只保存学习进度。

## 2. 方案比较

### 方案 A：全部放入 localStorage

实现最快，但容量、事务、查询和故障恢复能力都弱；大词库和 Excel 导入容易产生整块 JSON 重写。仅适合一次性原型，不推荐。

### 方案 B：IndexedDB（推荐）

浏览器原生支持结构化本地数据、索引和事务，无需启动额外服务。需要增加数据访问层和升级机制，但在 GitHub Pages 静态网页边界内长期成本最低。

### 方案 C：本地 SQLite/桌面服务

一致性和工具能力最好，但会把当前纯网页变成需要额外运行时的程序；超出 GitHub Pages + 浏览器本地数据的当前边界，不采用。

## 3. 首期支持格式

- CSV：最低依赖、易制作，必须支持 UTF-8；
- JSON：用于完整备份、恢复和程序化生成；
- XLSX：支持选择 Sheet 和字段映射，适合现有教材词表工作流；不执行公式或宏，只读取单元格显示值。

当前模板字段：

- 必填：`word`、`meaning`；
- 可选：`answer`、`phonetic`、`example`、`unit`、`source`、`tags`。

`answer` 缺省时等于 `word`。导入页可手动映射 `word`、`meaning`、`phonetic`、`example`；其他字段需使用标准名或已支持别名。`grade`、`book`、`notes`、`sourceUrl`、`license` 尚未写入当前数据层。模板、别名和长度限制见 `vocabulary-import-format-v1.md`。

## 4. 数据模型

当前 `Library`：`id`、`name`、`description`、`contentVersion`、创建/更新时间和归档时间。

当前 `VocabularyEntry`：`id`、`libraryId`、`word`、`normalizedWord`、`answer`、`meaning`、`phonetic`、`example`、`unit`、`unitLabel`、`sourceType`、`tags`、启用状态和创建/更新时间。

`id` 使用稳定 UUID，学习记录始终引用此值。同次导入以 `normalizedWord + meaning` 判断完全重复；相同英文但释义不同可以保留。“跳过”保留第一次出现，“覆盖”使用后项内容并保留该重复项首次生成的 ID。每次导入新建词库，不跨已有词库自动合并。

## 5. 导入流程

1. 选择文件；XLSX 可选择 Sheet。
2. 自动识别标准/别名表头，并让用户确认四个核心字段映射。
3. 展示前 20 行预览和总行数。
4. 校验必填项、长度、空行、非法类型和重复项。
5. 用户选择“跳过”或“后项覆盖”重复处理策略。
6. 在一个 IndexedDB 事务中写入词库和词条；任一关键错误则整体回滚。
7. 显示导入报告：成功、跳过、失败数量；失败行信息由数据层保留。
8. 新词库立即加入词库选择，但不自动开始学习；界面提示导出完整备份。

## 6. 与学习历史的关系

- 导入后不改变既有会话和统计。
- 词条更新不得改变 `id`；修改拼写时需明确提示可能影响历史解释。
- 已产生历史记录的词条禁止硬删除，只能停用或归档。
- 词库归档后不参与新会话，但历史记录仍显示当时词形和释义快照。
- 导出备份必须同时包含自定义词库、学习记录、数据版本和导出时间。

## 7. 安全与容错

- 已限制单文件 10 MB、任一 Sheet 20,000 行。
- 不执行 Excel 宏、公式、HTML 或导入文本中的脚本。
- 页面渲染继续使用 React 文本节点，不使用未经清洗的 `innerHTML`。
- 导入事务失败不留下半套词库；数据库未来升级前自动备份仍属后续增强。
- 外部来源的教材词表仍需记录来源和授权状态，导入能力不等于内容已获授权。

## 8. 实施结果与后续

### P1：CSV/JSON 最小版（已完成）

词库列表、项目文档模板、字段校验、预览、去重、IndexedDB 保存、选择学习、JSON 备份恢复均已完成。页面内“一键下载空白模板”尚未实现，模板当前位于 `docs/templates/`。

### P2：XLSX 与高级维护（部分完成）

已完成 Sheet 选择、字段映射、重复覆盖/跳过和词库归档；逐词编辑、跨既有词库合并及完整行级错误摘要仍未实现。

### P3：稳健长期版（部分完成）

已完成事务回滚路径、1,000 词测试、完整备份版本校验和损坏备份拒绝；数据库版本迁移矩阵、接近上限的大文件人工性能和冲突审计仍待后续。

## 9. 验收标准

- 自动化导入 1,000 个词后重新读取，词库、词条和唯一 ID 完整；真实 1,000 词 CSV/XLSX 人工导入仍建议抽验；
- 重复策略结果与预览一致；无效文件不会修改数据库；
- 自定义词库可参与现有随机、错词和未学词选词逻辑；
- 归档词库不会破坏历史会话；
- 备份往返测试可恢复词库与进度，损坏版本不会覆盖现有数据；
- 桌面浏览器与 390×844 响应式视口通过；真实 iPhone Safari 仍待验收，不安排 WKWebView 或原生 iOS 验收。
